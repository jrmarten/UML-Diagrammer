Requirement Analysis

  1. Introduction

    1. Purpose of the System
      The purpose of this system is to facilitate writing Java code through the modeling of class structures in UML.
    2. Scope of the System
      This system is for Java programmers and program designers and developers.
    3. The success criteria of the project
      To create a functioning program which can diagram UML, and translate UML into Java code for an easy and useful utilization by any programmer.
  1. Proposed System

    1. Functional Requirements
      2. A user must be able to:
      • create and edit UML diagrams
      • read and write diagrams to a file
      • generate Java code from UML diagrams
      • make UML templates

    1. Nonfunctional Requirements
      1. Usability
        • This application should look native to the Windows or Mac based on the operating system being used.
        • There should be a maximum of two steps to create any UML element
        • The program will have a Single Document Interface.
      2. Reliability
        • The program will report any errors to the user instead of crashing, if possible.
      3. Performance
        • Should take the User no longer than 3 seconds to save a document.
        • Adding GUI elements should be instantaneous.

     

  2. Use Case Model

      1. Create UML Class Elements
        1. Priority level: High
        2. Participating actors
          • User
          • System
        3. Flow of events
          1. User actively clicks anywhere on the document
          2. System creates UML Class at the location where the user click occurred
        4. Entry conditions
          • User presses Edit/Create/UML Class
          • User presses Right-click/Create/UML Class
        5. Exit conditions
        • UML Class is created
      2. Edit UML Class Elements
        1. Priority level: Medium
        2. Participating actors
          • User
          • System
        3. Flow of events
          1. System prompts User with an 'Edit Element' window
        4. Entry conditions
          • User Right-click Element/Edit/Add Attribute
          • User clicks Edit/Add/Attribute

      3. Create Association
        1. Priority level: High
        2. Participating actors
          • User
          • System
        3. Flow of events
          1. User will select which type of association
          2. User will select one UML element
          3. User will select a second UML element that is not the first
          4. System creates a link between the two elements
        4. Entry conditions
          • User Clicks Edit/Create/Association
          • User Right clicks first element
        5. Exit conditions
          • User cancels
          • Link has been created

      4. Remove Association
        1. Priority level: High
        2. Participating actors
        3. User

        4. System
        5. Flow of Events
          1. User selects an association
          2. System unlinks the elements
        6. Entry Conditions
          • User Clicks Edit/Association
        7. Exit Conditions
          • Link has been removed

    1. Open File
      1. Priority Level: Medium
      2. Participating actors
        • User
        • System
      3. Flow of events
        1. System prompts User for a filename that exists on the System
        2. User provides/selects filename
        3. System opens a new document with the contents of the file
      4. Entry conditions
        • User presses File/Open
      5. Exit conditions
        • User chooses filename
        • User cancels

    1. Write File
      1. Priority level: Medium
      2. Participating actors
        • User
        • System
      3. Flow of events
        1. System prompts User for new or existing filename
        2. User enters a filename
        3. System saves document to a file on the system
      4. Entry conditions
        • User presses File/Save
        • User presses File/Save as...
        • User presses Save button
      5. Exit conditions
        • User specifies a file
        • User cancels

         

    2. Generate Java Skeletons
      1. Priority level: High
      2. Participating actors
        • User
        • System
      3. Flow of events
        1. System creates Java files
      4. Entry conditions
        • User presses File/Export/Java Project
        • User presses Right click/Export/Java Porject
      5. Exit conditions
        • System is done writing Java files

    3. Create UML Template
      1. Priority level: Low
      2. Participating actors
        • User
        • System
      3. Flow of events
        1. System prompt user for name of the template
        2. System saves current document into templates file
      4. Entry conditions
        • Document is not empty
        • User presses File/Templates/Create
      5. Exit conditions
        • System finishes saving file

      1. Use UML Template
        1. Prioity level: Low
        2. Participating Actors
          • User
          • System
        3. Flow of Events
          1. Prompt user for which template to fill the document
          2. If current document is not empty, system prompts the user if he/she wants to continue
          3. If current document is not empty, clear it
          4. Fill current document with template
        4. Entry Conditions
          • User presses File/Templates/Open
        5. Exit Conditions
          • User decides to not continue
          • System finishes loading template

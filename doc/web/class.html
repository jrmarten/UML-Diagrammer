Class

Classes can take the form of Concrete Classes, Abstract Class or Interfaces. Concreate Classes and Abstract Classes may contain Attributes. All three type may contain Methods.


Attributes can be final or static. When inputing a textual representation of an Attribute into UML-Diagrammer, it can take two forms.

One is java like: public static final Object[][] mapping

The other is UML like: + static final mapping : Object[][]


Methods can be static xor abstract. Like attributes Methods can take two forms.

First is java like: public static void main ( String[] args )

Second is UML like: + static main ( args : String[] ) : void
